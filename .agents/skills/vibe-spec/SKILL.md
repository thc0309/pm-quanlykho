---
name: vibe-spec
description: Create or update SPEC.md for a new feature, greenfield project, or existing codebase onboarding before implementation.
---

Use `spec-driven-development`.

Start by understanding what the user wants to build. Ask only material clarifying questions about:

1. Objective and target users.
2. Core features and acceptance criteria.
3. Tech stack preferences and constraints.
4. Boundaries: always do, ask first, never do.

Then create or update `SPEC.md` with objective, commands, project structure, code style, testing strategy, and boundaries.

For an existing codebase, inspect the current implementation first and write `SPEC.md` to reflect reality before asking about new work.

Stop after presenting the spec and ask for confirmation before planning.
